Mock Interviews

Mock · Agentic AI Round

A 45-minute agent round. Drills the loop, control patterns, reliability and safety — and whether you know when NOT to build an agent.

Technical deep-dive

Overview

Tests engineering judgment around the loop, not prompt cleverness. Pair with Agentic AI.

Interviewer script

“Describe an agent you built.” → “How does tool calling actually work?” → “How does it not loop forever / call the wrong tool?” → “Where's prompt injection a risk?” → “When would a workflow be better?” Push on reliability and safety twice each.

Timing plan

Questions

QHow does tool calling work end to end?Intermediatetechnical deep-diveSeniorfreq: highBoth
Model answer

The model is given tools with input schemas. It outputs a structured request to call one. Your code runs the real function and returns the result into the conversation. The model then decides the next step or finishes. Repeat, with a step limit.

The model only proposes the call — your runtime executes it.

Strong senior-level answer

A strong answer frames it as 'model proposes, runtime disposes': the runtime validates arguments, keeps tools least-privilege, and bounds the loop.

Likely follow-ups
  • What stops infinite loops?
Red-flag answer pattern
Thinking the LLM itself runs the tool/code.
QHow do you make an agent reliable in production?Advancedtechnical deep-diveSeniorfreq: highBoth
Model answer

Bound the loop (max steps) and set a token/cost budget. Validate every tool call against its schema. Make tools least-privilege and idempotent. Add timeouts and retries. Guard inputs and outputs.

Strong senior-level answer

A strong answer adds checkpointed state (retry one step, not the whole run), human-in-the-loop on risky actions, and treating tool outputs as untrusted (injection risk).

Likely follow-ups
  • Where do you put human-in-the-loop?
Red-flag answer pattern
No bound on the loop — trusting the model to stop on its own.
QWhen would you build a workflow instead of an agent?Intermediatesystem designSeniorfreq: mediumBoth
Model answer

When the steps are known in advance. A deterministic workflow is then cheaper, faster, easier to test, and more reliable than a free-running agent loop.

Strong senior-level answer

A strong answer defaults to the least-agentic solution that works and only uses a full agent when the path is genuinely dynamic.

Likely follow-ups
  • Give an example.
Red-flag answer pattern
Reaching for an agent for everything.

Expected answer signals (strong)

  • 'Model proposes, runtime disposes'
  • Bounds the loop + budgets
  • Names indirect injection via tool output
  • Knows when a workflow wins

Weak answer signals

  • Thinks LLM executes tools
  • No loop bound
  • Only direct injection
  • Over-uses agents
