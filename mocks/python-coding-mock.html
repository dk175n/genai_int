Mock · Python Coding Round

A 45-minute applied coding round — not LeetCode, but realistic GenAI-engineering tasks: call an LLM with structured output + retry, build a small retrieval function, write the eval loop, handle async/batching.

Coding

Overview

Tests production Python for AI: clean code, error handling, testing, and the LLM-integration patterns from Prompting & LLM APIs.

Interviewer script

Task 1 (15 min): function that calls an LLM, forces structured output, validates, retries on invalid. Task 2 (15 min): implement top-k retrieval over precomputed embeddings + a recall@k eval. Task 3 (10 min): make N calls concurrently with a rate limit. Debrief (5 min). Watch for tests, types, and error handling.

Questions

QImplement a function that returns validated JSON from an LLM, retrying on schema failure.IntermediatecodingSeniorfreq: highBoth
Model answer

Pydantic schema + tool/structured call + validate; bounded repair loop feeding the error back; low temperature; log failure rate.

Strong senior-level answer

Treats schema as source of truth; bounds retries; adds a metric.

Likely follow-ups
  • What if it never validates?
  • How do you test it?
Red-flag answer pattern
Regex on prose, no validation.
Related hands-on task / project

Prompting hands-on.

QCompute recall@k for a retriever given labeled query→relevant-doc pairs.IntermediatecodingMid–Seniorfreq: mediumBoth
Model answer

For each query, fraction of relevant docs in top-k; average. Handle ties and empty results.

Strong senior-level answer

Vectorizes, adds edge-case handling and a quick test.

Likely follow-ups
  • MRR too?
Red-flag answer pattern
Off-by-one / no edge cases.
QMake 100 LLM calls concurrently without exceeding a rate limit.AdvancedcodingSeniorfreq: mediumBoth
Model answer

asyncio + a semaphore (or token-bucket) to cap concurrency; gather with error handling; backoff on 429.

Strong senior-level answer

Bounds concurrency, retries with jittered backoff, surfaces partial failures.

Likely follow-ups
  • How do you handle 429s?
Red-flag answer pattern
Fires all 100 at once.

Expected answer signals (strong)

  • Writes a test
  • Uses types + Pydantic
  • Handles errors/retries/edge cases
  • Explains tradeoffs while coding

Weak answer signals

  • No validation/tests
  • Ignores rate limits/errors
  • Silent failure paths
  • Can't explain choices
