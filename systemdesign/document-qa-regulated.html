System Design

Design · Document Q&A for Legal / Finance / HR

RAG for high-stakes, regulated domains: precise citations, strict abstain, auditability, and zero tolerance for confident fabrication. Accuracy and traceability dominate.

High-stakes

The brief

Same RAG skeleton, but the cost of a wrong answer is legal/financial. Design priorities flip to precision, mandatory citations, conservative abstain, and full auditability — speed and cost are secondary.

Clarifying questions to ask first

  • What's the consequence of an incorrect answer (legal liability)?
  • Is human verification required before action?
  • Retention/audit and data-residency requirements?
  • How precise must citations be (clause/line level)?

Functional requirements

  • Answer only with precise, clause-level citations
  • Abstain readily; never guess
  • Full audit log of question, context, answer
  • Versioned documents (which version was cited)

Non-functional requirements

  • Very high faithfulness; near-zero unsupported claims
  • Auditability/retention compliance
  • Data residency / on-prem option
  • Reproducibility (same Q → same cited answer)

Architecture

Question
Retrieve (precise chunks)

Audit log

Data ingestion layer

Version-aware ingestion (cite the exact document version); fine-grained chunks for clause-level citation; rich metadata (jurisdiction, effective date).

Retrieval layer

Hybrid with sparse weighted up (exact legal/financial terms matter); rerank; return precise spans for citation.

Model / orchestration layer

Very strict grounding: answer only from context, quote + cite the clause, abstain otherwise. Low temperature for reproducibility.

Safety layer

Mandatory citation presence; faithfulness gate blocks unsupported claims; PII/confidentiality controls; access control by matter/client.

Evaluation layer

Expert-labeled golden set; faithfulness + citation-correctness as hard gates; track unsupported-claim rate (target ~0).

Observability layer

Full immutable audit trail (question, retrieved version, answer, citations); monitor abstain rate and citation accuracy.

Cost & latency tradeoffs

LeverEffectTradeoff
Larger model↑ accuracy↑ cost — justified here
More verification passes↑ faithfulness↑ latency — acceptable

Failure modes

  • Confident fabrication → strict grounding + faithfulness gate + abstain
  • Citing wrong version → version-aware ingestion
  • Missing exact term → weight sparse retrieval
  • Audit gap → immutable logging

How to explain this design in 10 minutes

Open by naming the stakes; emphasize precise citations, strict abstain, version-awareness and audit; treat faithfulness as a hard gate; accept higher cost/latency for accuracy; close with failure modes centered on fabrication.

Common interviewer interruptions & how to answer

Interviewer says…How to answer
“What if retrieval returns nothing relevant?”Coverage gate → abstain or escalate; never force an answer. Measure abstain-correctness.
“This will be too expensive.”Cache + route to a smaller model for easy traffic; show cost/req and that quality held on eval.
“How do you know it's any good?”Golden set offline as a deploy gate + sampled online judging; split retrieval vs generation metrics.
“What about data the user shouldn't see?”Access control enforced at retrieval, not the prompt; per-tenant filters or separate indexes.
“What happens when the model provider goes down?”Timeouts, fallback model/provider, graceful degradation to cached/smaller answers.

Interview questions on this design (3)

QHow do you design RAG where a confident wrong answer is unacceptable?Seniorsystem designSenior/Stafffreq: mediumBoth
Model answer

Mandatory clause-level citations, a faithfulness gate that blocks unsupported claims, a conservative abstain policy, version-aware citations, and a full audit trail. Bias every tradeoff toward precision over coverage.

Strong senior-level answer

I'd add a verification pass (a second model checks each claim against its cited span) and human-in-the-loop before any action. I accept higher cost and latency and a higher abstain rate as the correct trade in a regulated domain.

Likely follow-ups
  • Would you ever let it answer without a citation?
  • How do you make it reproducible?
Red-flag answer pattern
Optimizing for coverage/UX in a domain where wrong answers create liability.
QHow do you handle document versions and effective dates in a legal/finance Q&A?Seniorsystem designSenior/Stafffreq: mediumBoth
Model answer

Version-aware ingestion: keep each document version with effective/expiry dates in metadata, cite the exact version used, and filter retrieval to the version in force for the query's context (e.g. 'as of' date).

Strong senior-level answer

A senior treats 'which version applied when' as a first-class requirement — citing the wrong version is as bad as a wrong answer in regulated domains — and keeps superseded versions queryable for audit.

Likely follow-ups
  • How do you answer 'what was the policy last year'?
Red-flag answer pattern
Indexing only the latest version and losing historical accuracy.
QWhy add a separate verification pass here, and how does it work?Advancedsystem designSenior/Stafffreq: emergingBoth
Model answer

A second model checks each claim in the answer against its cited source span and blocks/flags unsupported claims. In high-stakes domains the extra cost/latency is worth near-zero fabrication.

Strong senior-level answer

A senior accepts a higher abstain rate and latency as the correct trade, and routes anything that fails verification to human review rather than to the user.

Likely follow-ups
  • What's the latency/cost impact and is it acceptable?
Red-flag answer pattern
Skipping verification to keep latency low in a liability-heavy domain.
