QWrite a function that returns validated JSON from an LLM, retrying on failure.IntermediatecodingSeniorfreq: highBoth
Model answer

Approach:

  • Define the output shape as a Pydantic model.
  • Call the LLM forcing structured-output / tool-calling against that schema.
  • Validate the response; if it fails, retry a bounded number of times, passing the validation error back to the model.
  • Keep temperature low and log how often validation fails.
Strong senior-level answer

A senior makes the schema the source of truth, bounds the retries, and tracks the failure rate as a quality metric.

Likely follow-ups
  • What if it never produces valid output?
  • How do you test this?
Red-flag answer pattern
Using regex/string parsing on prose with no validation.
QMake 100 LLM calls concurrently without exceeding a rate limit.AdvancedcodingSeniorfreq: mediumBoth
Model answer

Use asyncio with a concurrency cap:

  • Wrap calls in asyncio.gather.
  • Use a Semaphore (or a token bucket) to limit how many run at once.
  • Catch errors so one failure doesn't sink the batch.
  • Retry on rate-limit (429) errors with exponential backoff.
Strong senior-level answer

A senior bounds concurrency, uses jittered backoff, and surfaces partial failures rather than silently dropping them.

Likely follow-ups
  • How do you handle 429 errors specifically?
Red-flag answer pattern
Firing all 100 requests at once with no limit.
QImplement recall@k for a retriever.IntermediatecodingMid–Seniorfreq: mediumBoth
Model answer

recall@k = of the documents that should have been found, what fraction appear in the top k results.

  • For each query: count how many of its relevant docs are in the top k, divide by total relevant.
  • Average across all queries.
  • Handle edge cases: ties, fewer than k results, queries with no relevant docs.
Strong senior-level answer

A senior writes a quick test and handles the edge cases cleanly.

Likely follow-ups
  • How would you add MRR?
Red-flag answer pattern
Off-by-one errors or ignoring edge cases.
QStream an LLM response and parse it safely.AdvancedcodingSeniorfreq: emergingBoth
Model answer
  • Stream tokens to the UI so the user sees progress.
  • But buffer the full response before acting on it for anything a program consumes.
  • Only validate/parse and take action on the completed message, not partial fragments.
Strong senior-level answer

A senior separates 'display to the human' from 'trust for the program' — guarding and validating the final, complete message.

Likely follow-ups
  • How do you apply a guardrail to a streamed answer?
Red-flag answer pattern
Acting on partial, unvalidated streamed output.
