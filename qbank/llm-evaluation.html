QHow do you evaluate a GenAI feature, before and after shipping?Advancedtechnical deep-diveSeniorfreq: highBoth
Model answer
  • Offline — run a curated 'golden' set of test cases before deploy, like a test suite, and block the release if quality drops.
  • Online — sample real production traffic and score it with the same checks, plus capture user feedback (thumbs up/down).
  • For RAG, split the metrics into retrieval quality and answer quality.
Strong senior-level answer

A senior runs evaluation as CI (no change ships without passing the golden set) and closes the loop — every production failure becomes a new test case.

Likely follow-ups
  • How big should the golden set be?
Red-flag answer pattern
'We manually check a few outputs.'

QHow do you trust an LLM-as-judge?Advancedtechnical deep-diveSeniorfreq: mediumBoth
Model answer

LLM-as-judge means using an LLM to score outputs. To trust it:

  • Give it a clear scoring rubric and low temperature.
  • Prefer comparing against a reference answer or pairwise comparisons.
  • Check its scores against a sample of human labels (calibration).
  • Watch for biases — it may favor longer answers or the first option.
Strong senior-level answer

A senior measures the judge's agreement with humans on a labeled sample, uses a different/stronger model as judge than the one being judged, and versions the judge prompt.

Likely follow-ups
  • What if the same model is judge and generator?
Red-flag answer pattern
Treating judge scores as ground truth with no calibration.

QWhat is a golden dataset, and how big should it be?Intermediatetechnical deep-diveMid–Seniorfreq: highBoth
Model answer

A golden dataset is a curated set of test inputs with expected answers or pass/fail criteria, used to measure quality and catch regressions.

  • Coverage matters more than size — 30–200 well-chosen cases beat thousands of near-duplicates.
  • Include happy paths, edge cases, adversarial inputs, and 'should say I don't know' cases.
  • Grow it from real production failures.
Strong senior-level answer

A senior tags cases by category (so they can see which type regressed) and keeps a permanent adversarial slice as a gate.

Likely follow-ups
  • How do you keep it from going stale?
Red-flag answer pattern
Thinking you need thousands of cases before you can start.
QOffline eval passes but users complain — what do you do?Advancedtechnical deep-diveSeniorfreq: mediumBoth
Model answer

It means your golden set doesn't reflect real usage. So:

  • Sample the failing production traffic and look at what actually went wrong.
  • Add those cases to the golden set.
  • Check whether the input distribution has drifted (users asking new kinds of questions).
Strong senior-level answer

A senior closes the loop online → golden set continuously, and monitors input drift so they catch the shift before users do.

Likely follow-ups
  • How do you detect drift without labels?
Red-flag answer pattern
Insisting it must be fine because offline passed.
