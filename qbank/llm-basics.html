QWhat is an LLM and how does it generate text?BeginnerscreeningMidfreq: highBoth
Why this is asked now

It's the foundation every other answer builds on; interviewers check you actually understand the mechanism.

Model answer

An LLM is a model that predicts the next word (token) in a sequence. It generates text by doing this repeatedly:

  • You give it some text (the prompt).
  • It outputs a probability for every possible next token.
  • It picks one, adds it to the text, and repeats until done.
  • It keeps no memory between calls — each request only knows what you send it.
Strong senior-level answer

A senior adds why this matters: because it's stateless, you must re-send chat history and any retrieved context every call; and because it predicts plausibility (not truth), quality is statistical — which is why we add retrieval, evaluation and guardrails.

Likely follow-ups
  • Why is it stateless?
  • Where does the 'knowledge' come from?
Red-flag answer pattern
Describing it as a database or search engine that 'looks up' answers.

QWhy do LLMs hallucinate?IntermediatescreeningSeniorfreq: highBoth
Why this is asked now

It connects directly to RAG, evaluation and guardrails — the topics interviewers care most about.

Model answer

Because the model generates the most plausible-sounding continuation, not necessarily the true one. When the likely text isn't the correct text, it still says it confidently.

You reduce it (you can't fully remove it) by:

  • Grounding answers in retrieved sources (RAG).
  • Telling it to cite sources and say 'I don't know' when unsure.
  • Lowering temperature for factual tasks.
  • Validating the answer against the source before showing it.
Strong senior-level answer

A senior treats hallucination as a measurable number (a 'faithfulness' or 'groundedness' score), monitors it in production, and fixes it with architecture and evaluation — not by adding 'please don't hallucinate' to the prompt.

Likely follow-ups
  • How would you measure hallucination?
  • What if retrieval finds nothing relevant?
Red-flag answer pattern
'You just tell it not to hallucinate in the prompt.'

QExplain temperature and top-p.BeginnerscreeningMidfreq: mediumBoth
Why this is asked now

A quick check that you've actually worked with the API knobs.

Model answer

Both control how random the model's output is.

  • Temperature — higher = more random/creative, lower = more focused/repeatable.
  • top-p — only sample from the most likely tokens that together cover p% of the probability.
  • Use low (0–0.3) for extraction, classification or JSON; higher (0.7–1.0) for brainstorming or creative writing.
Strong senior-level answer

A senior defaults to low temperature in production because it makes output reproducible and easy to evaluate, and only raises it where variety is genuinely the goal.

Likely follow-ups
  • Which would you use for JSON extraction and why?
Red-flag answer pattern
Using high temperature for structured/extraction tasks.
QWhat is the context window and why does it constrain design?Intermediatetechnical deep-diveSeniorfreq: highBoth
Why this is asked now

It directly shapes how you design RAG and chat memory.

Model answer

The context window is the maximum amount of text (tokens) the model can take in plus produce in a single call. It matters because:

  • Everything competes for that space — system prompt, chat history, and any retrieved documents.
  • You pay per token and latency grows as it fills up.
  • Very long contexts can lose accuracy in the middle ('lost in the middle').
Strong senior-level answer

A senior budgets the window deliberately — caps history, reranks retrieved context down to the few most relevant chunks — and knows that a bigger window isn't automatically better quality; relevance and placement matter more than raw size.

Likely follow-ups
  • What is 'lost in the middle'?
  • How do you decide how many chunks to include?
Red-flag answer pattern
Thinking the model remembers previous calls without you re-sending context.
