QDesign an enterprise knowledge assistant over our internal docs.Seniorsystem designSenior/Stafffreq: highBoth
Model answer

Drive it with a framework:

  • Clarify — which sources, permissions, freshness, scale, quality bar.
  • Requirements — functional (cited answers, follow-ups) and non-functional (latency, cost, no data leaks).
  • Architecture (layers) — ingestion → retrieval (with access control) → orchestration → safety → evaluation → observability.
  • Close with tradeoffs and failure modes.
Strong senior-level answer

A senior front-loads the non-functional bar (it drives every choice), enforces access control at retrieval, and treats evaluation and observability as design components — not afterthoughts.

Likely follow-ups
  • How do you keep the index fresh?
  • How do permissions work?
Red-flag answer pattern
'The user asks the LLM and it answers' — no retrieval quality, safety or evaluation.

QHow is GenAI system design different from classic system design?Intermediatesystem designSeniorfreq: mediumBoth
Model answer

It includes all the usual concerns (APIs, storage, caching, scaling) plus extra ones:

  • The model is probabilistic — correctness is statistical, not exact.
  • Retrieval quality and hallucination risk.
  • Prompt/version management.
  • Token cost and model latency dominate.
  • Safety, prompt injection, and evaluation/feedback loops.
Strong senior-level answer

A senior makes evaluation and observability first-class parts of the architecture and designs the feedback loop (production failures → eval set → improvement) in from the start.

Likely follow-ups
  • Where does evaluation live in the architecture?
Red-flag answer pattern
Treating it like a normal CRUD service design.
QDesign a multi-tenant production RAG platform (used by many teams).Seniorsystem designStafffreq: emergingBoth
Model answer

It's a shared platform, not one app:

  • Shared ingestion, retrieval, and a model gateway (routing, caching, fallback, rate limits).
  • A shared evaluation harness and guardrail layer every app inherits.
  • Per-tenant isolation (separate namespaces) and cost attribution.
  • Central observability.
Strong senior-level answer

A senior makes standards executable — an app can't onboard without passing baseline eval and inheriting guardrails — and keeps the gateway highly available since it's the critical path.

Likely follow-ups
  • How do you attribute cost per tenant?
  • How do you stop a shared cache leaking across tenants?
Red-flag answer pattern
One shared index for all tenants with no isolation story.
