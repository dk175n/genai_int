Project · Research Assistant Agent

A multi-step agent that plans, searches/reads multiple sources, and synthesizes a cited answer. Showcases planner-executor control and source attribution.

Planner-executor

What this project proves in an interview

You can orchestrate multi-step reasoning with tools while keeping cost bounded and outputs attributed.

Architecture

Question
Plan sub-questions
Search + read sources
Synthesize with citations
Self-check

Tech stack options

ConcernTool
ControlPlanner-executor / LangGraph
ToolsWeb/search + fetch + read
OutputSynthesis with citations

API design

POST /research → {question} ⇒ {report, sources:[...], steps:[...]}.

Data flow

Question → plan sub-questions → for each: search + fetch + extract → synthesize with citations → self-critique for unsupported claims → return report + sources.

Evaluation plan

Rubric-based: coverage, correctness, citation validity (do sources support claims?), and cost/steps per query.

Failure modes

  • Unbounded exploration → step/cost budget
  • Unsupported synthesis → require citations + self-check
  • Injection from fetched pages → untrusted content

Demo script

Ask a real question → show the plan, the sources used, and the cited synthesis. Show the step budget kept cost bounded.

Questions the interviewer may ask about this project (4)

QHow would you explain the Research Assistant project in an interview?IntermediatebehavioralSeniorfreq: highBoth
Model answer

Lead with the problem and the metric: “It decomposes a question, gathers and reads sources, and writes a cited synthesis.” Then the architecture in one breath, then the one hard tradeoff you made and how you evaluated it.

Strong senior-level answer

I structure it as: problem → users → architecture → the hardest decision and why → how I measured success → what I'd do next. I keep it to 2 minutes and let the interviewer pull threads.

Likely follow-ups
  • What was the hardest part?
  • What would you change with more time?
  • How did you know it worked?
Red-flag answer pattern
Listing technologies with no problem, metric, or tradeoff.
QHow do you keep a multi-step research agent's cost and quality under control?Advancedtechnical deep-diveSeniorfreq: mediumBoth
Model answer

Plan up front (planner-executor) so steps are predictable, set a step/token budget, require citations and run a self-critique to drop unsupported claims, and treat fetched content as untrusted.

Strong senior-level answer

I make the plan explicit and bounded so cost is predictable, and I add a verification pass that checks each claim against a cited source — synthesis without attribution is where these agents quietly hallucinate.

Likely follow-ups
  • Why planner-executor over ReAct here?
Red-flag answer pattern
Open-ended ReAct with no budget or citation requirement.
QHow do you stop the research agent from hallucinating a synthesis?Advancedtechnical deep-diveSeniorfreq: mediumBoth
Model answer
  • Require every claim in the synthesis to cite a fetched source.
  • Run a self-critique pass that flags unsupported claims and removes them.
  • Treat fetched web content as untrusted (injection risk).
Strong senior-level answer

A senior adds a verification step that checks each claim against its cited source — synthesis without attribution is exactly where these agents quietly fabricate.

Likely follow-ups
  • How is fetched content an injection vector?
Red-flag answer pattern
Trusting the synthesis because it 'sounds authoritative'.
