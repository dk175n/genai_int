Project · Email Triage Agent

Classify and route incoming emails (category, urgency, suggested action) with structured output and a human-in-the-loop for low-confidence cases.

Project

What this project proves in an interview

You can ship a practical automation with structured output, confidence handling and human-in-the-loop — the bread-and-butter of applied GenAI.

Architecture

Email
Classify → structured output
Confidence gate
Route / draft reply / escalate

Tech stack options

ConcernTool
ExtractionTool calling + Pydantic schema
RoutingRules on structured fields
HITLQueue low-confidence for review

API design

POST /triage → {email} ⇒ {category, urgency, needs_human, draft_reply?}.

Data flow

Email → structured classification (category, urgency, confidence) → if confidence low or action risky → human queue; else auto-route + optional draft reply.

Evaluation plan

Labeled emails → measure classification accuracy, and precision on the 'auto-handle' decisions (you care most about not auto-acting wrongly).

Failure modes

  • Over-confident misclassification → confidence gate + HITL
  • Sensitive content auto-handled → policy guard
  • Format drift → schema validation

Demo script

Feed varied emails → show structured decisions; a low-confidence one routes to human; a clear one gets an auto draft.

Problem & users · Architecture · Quickstart · Design decisions · Evaluation · Limitations & next steps.

Interview explanation template

“An email triage agent that outputs structured decisions and only auto-acts when confident, escalating the rest. It's about safe automation, not full autonomy.”

Questions the interviewer may ask about this project (4)

QHow would you explain the Email Triage project in an interview?IntermediatebehavioralSeniorfreq: highBoth
Model answer

Lead with the problem and the metric: “It classifies and routes emails with structured output, auto-handling confident cases and escalating the rest.” Then the architecture in one breath, then the one hard tradeoff you made and how you evaluated it.

Strong senior-level answer

I structure it as: problem → users → architecture → the hardest decision and why → how I measured success → what I'd do next. I keep it to 2 minutes and let the interviewer pull threads.

Likely follow-ups
  • What was the hardest part?
  • What would you change with more time?
  • How did you know it worked?
Red-flag answer pattern
Listing technologies with no problem, metric, or tradeoff.
QHow do you decide when to auto-act vs escalate to a human?Intermediatetechnical deep-diveSeniorfreq: mediumBoth
Model answer

Use a confidence signal and risk of the action: auto-handle high-confidence, low-risk cases; escalate low-confidence or high-impact ones. Tune the threshold on labeled data against the cost of a wrong auto-action.

Strong senior-level answer

I optimize for precision on the auto-handled slice because a wrong automated action is costlier than asking a human. I monitor the escalation rate and the auto-action error rate as live metrics.

Likely follow-ups
  • How do you get a calibrated confidence?
Red-flag answer pattern
Auto-acting on everything for a nicer demo.
QHow do you evaluate the triage agent?Intermediatetechnical deep-diveSeniorfreq: mediumBoth
Model answer

Labeled emails → measure classification accuracy, and especially precision on the auto-handled slice (a wrong auto-action is the costly error). Track escalation rate too.

Strong senior-level answer

A senior optimizes for precision on auto-actions over raw accuracy, because the business cost of a wrong automated reply outweighs an unnecessary escalation.

Likely follow-ups
  • Why precision over accuracy here?
Red-flag answer pattern
Reporting only overall accuracy.
QHow would you make this safe for sensitive emails (legal, HR)?Advancedsystem designSeniorfreq: mediumBoth
Model answer
  • Detect sensitive categories and force human review (never auto-act).
  • Redact PII before sending to the model and in logs.
  • Policy guard on any drafted reply before it leaves.
Strong senior-level answer

A senior designs the confidence/risk gate so high-impact categories always escalate regardless of model confidence.

Likely follow-ups
  • How do you redact PII without losing context?
Red-flag answer pattern
Auto-replying to sensitive emails to boost automation rate.

Build & ship checklist

  • Structured output + validation
  • Confidence/risk gate + HITL
  • Accuracy + auto-action precision eval
